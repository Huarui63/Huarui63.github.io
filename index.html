bat脚本入门1--基础知识

发表于 2017-01-05 |
问题描述

今天在uzhou的带领下，尝试用批处理脚本解决“使用有道云笔记编辑md文件，每次都要重复进行另存为操作”的问题。

我的理解

之前接触的脚本语言有shell，还有最近开始学习的python。
其实DOS批处理脚本语言我想所有学习计算机的人多多少少都接触过：在开始菜单输入cmd，打开一个DOS命令行窗口，事实上，在这个窗口下输入的命令都是bat脚本。只不过对它的理解太局限了，没有意识到其实自己就是在用批处理脚本语言。当我们把一批命令写到一个.bat文件中，然后执行这个bat文件，其实就是按行执行这些命令，省的自己每次一遍一遍地敲命令行了。

名词解释

bat：batch的简写，意思是：一批、成批作业。也是批处理的由来。

解决方案

在对一些基本的bat命令进行学习之后，我一开始选择的是将有道云笔记的文件存放路径下的所有.md文件都复制到hexo的_posts文件夹下，但是之后通过实践，发现出现将一些不需要的md文件复制过去的问题，所以还是决定将指定的md文件复制到hexo文件夹下。
代码如下:
@echo off set /p var=input the file name you want to update: for /r D:\BLOG\youdaoyunNOTE\18202789862@163.com %%i in (%var%.md) do if exist %%i copy %%i D:\BLOG\hexo\source\_posts pause

问题解决

期间出现了不能复制的问题，原因是没有进到相应的路径（D:\…）下面，仍然在默认路径（桌面）下。

操作系统学习笔记1--UMA、NUMA、SMT等概念问题

发表于 2017-01-02 |

1.multi-core

与多核计算机对应的是单核计算机，single-core计算机是指cpu芯片中只有一个计算单元ALU的计算机。而多核架构是指CPU芯片中有多个核（core），也就是有多个ALU的计算机。根据计算机的这些核之间的数据传输、存储等关系的不同，又可以把多核计算机分成UMA、NUMA等类别。

2.multi-kernel

是一种操作系统架构，把计算机当作由各个独立的核组成的一个网络，核之间通过消息传递（message-passing）交流。Barrelfish是一个典型的multi-kernel操作系统。

3.multi-processor

目前我理解的多处理器架构多见于服务器。日常使用的计算机只有一个cpu，也就是只有一个处理器。而在服务器或者集群中，由于有处理大量数据和运算的需要，所以需要多处理器。根据维基百科，multiprocessing是：在一个计算机系统中，使用两个及以上的处理器单元（CPU）。

4.UMA

全名：Uniform Memory Access。根据维基百科：UMA是一种用于并行计算机的共享数据架构。UMA中的所有处理器共享物理存储，数据存取时间与位置无关。在UMA中，每个处理器也可以有一个私有的cache

5.NUMA

全名：Non-Uniform Memory Access，是multiprocessing中的一种计算机存储设计，在NUMA中，存储有local和非local之分。数据的存取时间与数据的存放位置有关。各个node之间形成互连，访问本地内存的速度远高于非本地存储，因此NUMA架构的一个重要问题就是如何减少不同node之间的信息交流。

6.SMP

有两个含义：shared memory multiprocessor和symmetricmultiprocesspor。对称处理器结构是指多个CPU对称工作，无主次或者从属关系。各CPU共享相同的物理内存，每个CPU访问内存中的任何地址所需的时间是相同的，因此SMP也被成为UMA。因为所有的资源都是共享的，所以smp的扩展（scale）能力非常有限。

C语言的编译过程以及gcc语法

发表于 2016-12-12 |

由一个C源程序生成一个可执行文件要经过一下几个步骤：

1. 编译预处理

这个步骤主要是对源代码中各种以“#”开头的指令进行处理。如#include``#define等宏定义、条件编译、头文件包含指令。

2. 编译、优化

检查代码中的语法错误，并编译生成汇编文件(.s/.asm)

3. 汇编

将汇编代码翻译成目标文件(.o/.a/.ko/.obj)，也就是二进制代码。目标文件一般包括代码段和数据段（存放全局变量等数据）

4. 链接

将所有.o文件链接到一起，生成可执行文件(.exe)

GCC基本语法

  • gcc -c

gcc 后面是操作对象，-c或者-o后面是输出文件的名字
-c means “just Compile,no Make”
只编译，不连接

例子：gcc test.c 生成可执行文件a.out
gcc -c test.c输出test.o文件

  • gcc -o

指定生成的输出文件

例子：gcc test.c -o test 生成可执行文件test，-o后面是输出文件的名字
